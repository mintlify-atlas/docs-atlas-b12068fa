## Overview

ACE-Step 1.5 is a highly efficient open-source music foundation model that brings **commercial-grade generation to consumer hardware**. It achieves quality beyond most commercial music models while remaining extremely fast—under 2 seconds per full song on an A100 and under 10 seconds on an RTX 3090.

<Note>
ACE-Step 1.5 runs locally with **less than 4GB of VRAM** and supports lightweight personalization through LoRA training from just a few songs.
</Note>

## What Makes ACE-Step 1.5 Unique?

### Hybrid Architecture

At its core lies a novel hybrid architecture where the Language Model (LM) functions as an **omni-capable planner**:

- Transforms simple user queries into comprehensive song blueprints
- Scales from short loops to 10-minute compositions
- Synthesizes metadata, lyrics, and captions via Chain-of-Thought
- Guides the Diffusion Transformer (DiT) with precise instructions

<Tip>
The alignment between LM and DiT is achieved through **intrinsic reinforcement learning**, eliminating biases from external reward models or human preferences.
</Tip>

## Core Features

### Generation & Control

| Feature | Description |
|---------|-------------|
| **Reference Audio Input** | Use reference audio to guide generation style |
| **Cover Generation** | Create covers from existing audio |
| **Repaint & Edit** | Selective local audio editing and regeneration |
| **Track Separation** | Separate audio into individual stems |
| **Multi-Track Generation** | Add layers like Suno Studio's "Add Layer" feature |
| **Vocal2BGM** | Auto-generate accompaniment for vocal tracks |

### Metadata & Configuration

| Feature | Description |
|---------|-------------|
| **Metadata Control** | Control duration, BPM, key/scale, time signature |
| **Simple Mode** | Generate full songs from simple descriptions |
| **Query Rewriting** | Auto LM expansion of tags and lyrics |
| **Audio Understanding** | Extract BPM, key/scale, time signature & caption from audio |
| **LRC Generation** | Auto-generate lyric timestamps for generated music |

## Hardware Requirements

<Warning>
While ACE-Step 1.5 can run on GPUs with as little as 4GB VRAM, performance and features scale with available memory.
</Warning>

### Minimum Requirements

| Component | Requirement |
|-----------|-------------|
| **Python** | 3.11-3.12 (stable release) |
| **GPU** | CUDA GPU recommended |
| **VRAM** | ≥4GB for DiT-only mode, ≥6GB for LLM+DiT |
| **Disk Space** | ~10GB for core models |

### Supported Platforms

- **NVIDIA GPUs**: CUDA 12.8+ (Windows, Linux)
- **AMD GPUs**: ROCm 6.0+ (Linux), ROCm 7.2 (Windows with Python 3.12)
- **Intel GPUs**: XPU support (Windows, Linux)
- **Apple Silicon**: MPS/MLX backend (M1/M2/M3/M4)
- **CPU**: CPU-only mode for inference (not recommended for training)

## Model Selection Guide

<Tip>
ACE-Step automatically selects the best configuration for your GPU. All settings (LM model, backend, offloading, quantization) are tier-aware and pre-configured.
</Tip>

| Your GPU VRAM | Recommended LM Model | Backend | Notes |
|---------------|---------------------|---------|-------|
| **≤6GB** | None (DiT only) | — | LM disabled by default; INT8 quantization + full CPU offload |
| **6-8GB** | `acestep-5Hz-lm-0.6B` | `pt` | Lightweight LM with PyTorch backend |
| **8-16GB** | `acestep-5Hz-lm-0.6B` / `1.7B` | `vllm` | 0.6B for 8-12GB, 1.7B for 12-16GB |
| **16-24GB** | `acestep-5Hz-lm-1.7B` | `vllm` | 4B available on 20GB+; no offload needed on 20GB+ |
| **≥24GB** | `acestep-5Hz-lm-4B` | `vllm` | Best quality, all models fit without offload |

## Use Cases

ACE-Step 1.5 is designed for:

- **Music Artists & Producers**: Create original compositions, generate backing tracks, experiment with styles
- **Content Creators**: Generate royalty-free music for videos, podcasts, and streams
- **Game Developers**: Create dynamic music assets and adaptive soundtracks
- **Researchers**: Explore music generation, style transfer, and AI-assisted composition
- **Educators**: Teach music theory, composition, and AI applications in creative arts
