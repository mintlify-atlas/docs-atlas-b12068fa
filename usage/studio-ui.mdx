<Note>
The Studio UI is an **optional, experimental** feature. The default way to use ACE-Step remains the Gradio Web UI.
</Note>

The Studio UI provides an alternative HTML-based interface with a more structured, DAW-like layout for users who prefer a different visual approach to music generation.

## Overview

<Card title="Key Features" icon="star">
  - **Frontend-only**: Pure HTML/CSS/JavaScript interface
  - **REST API integration**: Communicates with existing ACE-Step API
  - **No backend changes**: Uses standard `/release_task` and `/query_result` endpoints
  - **Experimental**: Layout and features may evolve based on feedback
</Card>

## Quick Start

<Steps>
  <Step title="Start API Server">
    Launch the ACE-Step REST API server:
    
    ```bash
    uv run acestep --enable-api --port 8001
    ```
    
    Or use your usual API launch command.
  </Step>
  
  <Step title="Open Studio UI">
    Open `ui/studio.html` in a web browser:
    
    - **Double-click** the file
    - Or navigate to `file:///path/to/ACE-Step-1.5/ui/studio.html`
  </Step>
  
  <Step title="Configure API URL">
    Set the API base URL if needed (default: `http://localhost:8001`)
  </Step>
  
  <Step title="Generate Music">
    Enter prompt and options, then click **Generate**. The UI will poll for results and display audio when ready.
  </Step>
</Steps>

## Prerequisites

<Warning>
The Studio UI requires a running ACE-Step API server. It **cannot** function standalone.
</Warning>

**Required:**
- ACE-Step API server running (see [REST API](/usage/rest-api))
- Modern web browser with JavaScript enabled
- Network access to the API server

**Optional:**
- CORS configuration if UI and API are on different domains

## API Integration

The Studio UI uses standard ACE-Step REST API endpoints:

### Task Submission

```javascript
// Submit generation task
const response = await fetch(`${apiBase}/release_task`, {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    prompt: 'upbeat pop song',
    thinking: true,
    batch_size: 2
  })
});

const {data} = await response.json();
const taskId = data.task_id;
```

### Status Polling

```javascript
// Poll for results
const pollInterval = setInterval(async () => {
  const res = await fetch(`${apiBase}/query_result`, {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({task_id_list: [taskId]})
  });
  
  const {data} = await res.json();
  const status = data[0].status;
  
  if (status === 1) {
    // Success - display audio
    clearInterval(pollInterval);
  } else if (status === 2) {
    // Failed - show error
    clearInterval(pollInterval);
  }
}, 5000);
```

### Audio Download

```javascript
// Download generated audio
const audioUrl = `${apiBase}/v1/audio?path=${encodeURIComponent(audioPath)}`;
window.open(audioUrl, '_blank');
```

## Advanced Features

<Note>
Experimental features may change in future releases.
</Note>

### Batch Generation

Generate multiple variations:

1. Set batch size (1-8)
2. Configure seed settings
3. Click **Generate**
4. View all variations in results panel

### Model Selection

Switch between available DiT models:

1. Click **Models** dropdown
2. Select from available models
3. Model name appears in generation metadata

### File Upload

Upload source or reference audio:

1. Click **Upload** button
2. Select audio file (wav, mp3, flac)
3. File is sent to server for processing
4. Use in Cover, Repaint, or other audio tasks

### Custom Parameters

Access advanced DiT/LM settings:

1. Expand **Advanced Settings** panel
2. Configure inference steps, shift, timesteps
3. Adjust LM temperature, CFG scale
4. Parameters are sent with task submission

## Troubleshooting

<Tabs>
  <Tab title="Connection Issues">
    **Problem:** Cannot connect to API server
    
    **Solutions:**
    - Verify API server is running
    - Check API base URL is correct
    - Ensure firewall allows connections
    - Test `/health` endpoint directly
    - Review browser console for errors
  </Tab>
  
  <Tab title="CORS Errors">
    **Problem:** CORS policy blocks requests
    
    **Solutions:**
    - Run UI and API on same origin
    - Configure API server CORS headers
    - Use browser extension to disable CORS (dev only)
    - Serve UI through API server
  </Tab>
  
  <Tab title="Task Failures">
    **Problem:** Generation tasks fail
    
    **Solutions:**
    - Check API server logs for errors
    - Verify all required parameters are set
    - Ensure model is initialized
    - Review task status response for error details
  </Tab>
  
  <Tab title="Audio Playback">
    **Problem:** Cannot play generated audio
    
    **Solutions:**
    - Check audio format is supported by browser
    - Verify audio file URL is accessible
    - Test direct download of audio file
    - Review browser console for errors
  </Tab>
</Tabs>

## Limitations

<Warning>
Current limitations of the experimental Studio UI:
</Warning>

- **Frontend-only**: No backend processing or model inference
- **API dependency**: Requires running ACE-Step API server
- **Limited features**: Not all Gradio UI features are implemented
- **Experimental status**: Interface may change significantly
- **No training**: LoRA training not available in Studio UI
- **Browser compatibility**: Best experience in Chrome/Firefox
